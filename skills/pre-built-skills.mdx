The catalogue ships with **250+ prebuilt Skills** covering the most common bioinformatics tools and workflows. Each Skill encodes how a tool or procedure should be run — parameters, input/output handling, interpretation — so the agent can apply it correctly without manual configuration.

<Tip>
  You do not need to browse the catalogue before starting an analysis. The agent suggests relevant Skills as you describe your question. Browse when you want to see what's available or pre-attach Skills before running.
</Tip>

Open the catalogue from the wrench icon in a project's right sidebar, then click **+ Add**. Search filters as you type; each card has a one-line description and a click-through for the full text.

## How to use Skills in a project

<Steps>
  <Step title="Open the Skills panel">
    Click the wrench icon in the project's right sidebar.
  </Step>
  <Step title="Attach what you need">
    Click **+ Add** to open the catalogue. Search or browse, then click **+ Add** on a card to attach it.
  </Step>
  <Step title="Toggle on the active ones">
    The panel header shows the active count (e.g. *"3 of 5 enabled"*). Toggle a Skill on; the next prompt picks it up.
  </Step>
  <Step title="Verify in the Steps list">
    When the agent consults a Skill it logs a step like *"Enable pydeseq2 Skill"*. Open the Steps list to confirm.
  </Step>
</Steps>

## A few tips

- **Don't over-attach.** Two or three Skills per project is usually enough. Twenty competing Skills produce muddier results, not better ones.
- **Read the description before attaching.** A Skill is a brief — read it to make sure it matches what you want.
- **Use workflow Skills for repeatable pipelines.** A workflow Skill is far more reliable than re-describing the same five-step procedure each time.
