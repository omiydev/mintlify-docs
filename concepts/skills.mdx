A Skill is a short note, written in plain English, that tells the agent how you want a particular kind of work done. It is not code. It is not a plugin. It is closer to the brief you would give a new lab member — concise, opinionated, and reusable.

When the agent runs an analysis, it reads the Skills you have attached to the project and treats them as guidance. Attached pydeseq2 Skill? The agent will use pydeseq2 for differential expression. Attached an in-house QC Skill? The agent will apply your lab's QC thresholds before reporting.

## How Skills work in a project

Skills are **per project**, not global. Each project has its own attached set with a toggle for each one. The agent reads only the active Skills.

<Steps>
  <Step title="Open the Skills panel">
    Click the wrench icon in the project's right sidebar. A panel opens listing every Skill attached to this project, with a toggle next to each.
  </Step>
  <Step title="Read the active count">
    The header shows how many Skills are enabled out of the total attached — for example, *"4 of 7 enabled"*. The agent applies only the enabled Skills.
  </Step>
  <Step title="Add Skills from the catalogue">
    Click **+ Add** to open the catalogue. Search or browse by category. Each result is a card with a one-line description; click for the full text.
  </Step>
  <Step title="Toggle as you go">
    Switching a toggle takes effect on the next prompt. The agent will show a Shell step like *"Enable pydeseq2 Skill"* in the Steps list when it consults it — proof the toggle worked.
  </Step>
</Steps>

<Tip>
  Start with two or three Skills per project, not twenty. The agent works best with focused guidance. Add more as the analysis develops.
</Tip>

## The catalogue

Galagos ships with **250+ prebuilt Skills** covering the most common bioinformatics tools and workflows. Categories include:

- **Genomics and pipelines** — alignment, variant calling, quantification, QC.
- **RNA-seq and single-cell** — DESeq2, pydeseq2, edgeR, Scanpy, Seurat.
- **Protein structure and design** — AlphaFold, AlphaFold-Database, ColabFold, RoseTTAFold.
- **Chemistry and small molecules** — RDKit, AutoDock Vina, ChEMBL.
- **Proteomics and metabolomics** — peak-finding, identification, quantification.
- **Drug discovery** — target search, docking, screening.
- **Clinical and medical** — clinical-variant interpretation, evidence linking.

Search live-filters as you type. Typing *"alpha"* returns AlphaFold, AlphaFold-Database, alphagenome, and other alpha-prefixed Skills.

## Workspace-level Skills

In a team workspace, owners and admins can configure Skills at the workspace level:

- **Pinned** Skills become defaults — they are pre-attached when a member creates a new project.
- **Required** Skills cannot be toggled off — useful for QC thresholds, citation policy, or privacy rules that apply across every project.

See [Workspaces](/concepts/workspaces) for the full picture.

## Save a conversation as a Skill

After you've run an analysis you want to repeat, save the conversation itself as a Skill. The agent extracts the approach, the parameters, and the interpretation logic from the chat and turns them into a reusable Skill.

You can do this two ways:

- **From the chat** — after a successful run, the agent often offers *"Save this as a Skill?"* as a suggestion. Click it.
- **Manually** — open the Skills panel and click **Create**. Describe in plain English what the Skill should do; the agent drafts it and asks you to confirm.

## Custom Skills in plain English

A custom Skill is just a short brief. Writing examples:

- *"Whenever I upload gene expression data, log-transform counts, filter out genes with zero reads in more than half the samples, and include both fold change and adjusted p-value in the output."*
- *"For every phylogenetic tree, root on the outgroup I tell you, colour branches by host species, and scale branch lengths to substitutions per site."*
- *"When I share a paper, extract the methods, the figures, and the supplementary tables. I don't need the introduction."*

You write the brief; the agent saves it as a Skill in the project. Click **Save to My Skills** to promote it from project-local to your personal library, where it appears for every project you create.

See [Custom Skills](/skills/custom-skills) for the full how-to.

## A few tips

- **Brief like a new lab member, not a computer.** Tell the Skill *why* and *when*, not just *what*.
- **One sentence is enough to start.** You can refine later.
- **Layer two or three per project.** Avoid attaching a Skill *just in case* — it changes how the agent works on every prompt.
- **Update Skills when your approach changes.** A six-month-old Skill that says *"use limma-voom"* may not be what you want next month. Edit it, or replace it.
